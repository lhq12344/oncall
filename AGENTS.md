# AGENTS.md

适用范围：仓库根目录及所有子目录（除非子目录存在更高优先级的 `AGENTS.md`）。

## 通用规则

- 统一使用中文回复。
- 优先最小改动，禁止顺手重构无关代码。
- 不要修改与当前任务无关的文件、接口或配置。
- 修改前先阅读相关文件并确认调用链。

## Go 代码规则

- 该项目使用Goframe web框架和Eino大模型应用框架，需要严格按照这些框架的功能来构造函数和agent及其方法
- 生成的方法或者函数给出功能、输入、输出等注释
- 保持现有代码风格与命名习惯。
- 新增逻辑优先复用已有组件，避免重复实现。
- 修改 Go 代码后，至少运行一次 `go test ./...`；若失败，先定位是否为本次改动引入。
- 非用户明确要求，不新增无关依赖或大型框架改造。

## 本项目约束

- 对话、上传、运维能力按既有 Agent 分层保持边界，不混用职责。
- 中断恢复（checkpoint/resume）相关逻辑修改时，必须保持流式接口行为一致。
- 涉及上下文记忆时，优先考虑 token 成本与上下文长度控制，避免把大段日志直接回灌模型上下文。

## 输出与变更说明

- 回答中必须说明改了什么、为什么改、影响范围。
- 涉及接口或行为变化时，明确列出兼容性影响。
