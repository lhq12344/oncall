# AGENTS.md

适用范围：`internal/agent` 及其子目录（更深层 `AGENTS.md` 可覆盖）。

## Agent 设计边界
- 保持单一职责：`dialogue` 处理对话，`knowledge` 处理知识上传/索引，`ops` 处理运维流程。
- Agent 之间传递结构化数据，避免依赖自然语言拼接作为唯一契约。
- 新增工具前先检查是否已有同类工具可复用。

## 提示词与工具调用
- 提示词改动需保证输出可解析（JSON 契约字段名保持稳定）。
- 工具描述与参数定义必须与实现一致，避免“提示词可用、代码不可用”。
- 高风险执行相关能力必须保留人工确认/中断恢复机制。

## 上下文与状态
- 会话记忆与工作流状态分离：用户会话走 `Session Memory`，内部流转走 `Graph State`/结构化状态。
- 避免把大日志和大对象直接塞入模型历史，优先摘要、裁剪、检索。

## 质量要求
- 修改 agent 或 tool 后，至少运行 `go test ./...`。
- 非必要不调整公开接口与路由定义。
